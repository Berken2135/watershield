# WaterShield вЂ” Backend

Water pollution tracking API using Sentinel-2 satellite data (Copernicus openEO), pre-computed data-science outputs, AI anomaly detection, and EU Water Framework Directive compliance reports.

---

## What it does

- **Live satellite analysis** вЂ” user draws a polygon on the map, backend fetches Sentinel-2 imagery and returns water pollution indices
- **Pre-computed European data** вЂ” serves WQI scores, risk levels, and 30-day forecasts for 30 European cities (generated by the data-science pipeline)
- **AI anomaly detection** вЂ” classifies water measurements as normal / anomaly / critical using EU WFD thresholds, with optional OpenAI enrichment
- **PDF report generation** вЂ” generates downloadable EU-style compliance reports via ReportLab + optional OpenAI summary

---

## Requirements

- Python 3.12 (not 3.13+, pydantic-core requires 3.12)
- A Copernicus Data Space account with client credentials

Install Python 3.12 on Mac:
```bash
brew install python@3.12
```

## Credentials

Create a `.env` file in `watershield/backend/`:

```
CDSE_USERNAME=your_email@example.com
CDSE_PASSWORD=your_password

CDSE_CLIENT_ID=sh-xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx
CDSE_CLIENT_SECRET=your_client_secret

OPENEO_URL=https://openeo.dataspace.copernicus.eu

# Optional вЂ” enables AI-enriched anomaly verdicts and PDF summaries
OPENAI_API_KEY=sk-...
OPENAI_MODEL=gpt-4o-mini
```

> The `.env` file is gitignored. Never commit credentials.
> All endpoints work without `OPENAI_API_KEY` вЂ” AI features degrade gracefully to deterministic fallbacks.


Or directly:

### Data вЂ” pre-computed data-science outputs

These endpoints serve files generated by the data-science pipeline (`data-science/data/outputs/`). Files are read at request time вЂ” a fresh pipeline run is immediately live without restarting the server.

```
GET /api/data/europe
```
Full GeoJSON FeatureCollection вЂ” 30 European cities + WrocЕ‚aw. Each feature includes `wqi_current`, `risk_level`, `risk_color`, `trend`, and `metrics`. Drop directly into MapLibre as a GeoJSON source.

```
GET /api/data/wroclaw
```
WrocЕ‚aw-only feature with real Waterly sensor data, anomaly count (last 30 days), and MPWiK turbidity.

```
GET /api/data/summary
```
Quick stats: `total_cities`, `risk_counts`, `avg_wqi_by_country`.

```
GET /api/data/forecast
```
30-day WQI forecast for WrocЕ‚aw (Odra River). Each entry: `date`, `wqi_predicted`, `wqi_lower`, `wqi_upper`, `risk_level`. Winner model: XGBoost (7d RMSE = 11.9).

```
GET /api/data/forecast-metrics
```
Model evaluation metrics for Prophet and XGBoost at 7d and 30d horizons.

### Analysis вЂ” live Sentinel-2 (active events)

> вљ пёЏ These endpoints process real satellite data via openEO and take **60вЂ“120 seconds** to respond. This is expected.

```
POST /api/analysis/water-quality
```
```json
{
  "geometry": {
    "type": "Polygon",
    "coordinates": [[[19.9, 50.0], [20.1, 50.0], [20.1, 50.2], [19.9, 50.2], [19.9, 50.0]]]
  },
  "start_date": "2024-06-01",
  "end_date": "2024-06-30",
  "max_cloud_cover": 30
}
```
Returns a per-date timeline of 4 pollution indices (NDWI, chlorophyll, turbidity NDTI, CDOM) + pollution score (0вЂ“100) + alert level + summary. Accepts `Polygon` or `MultiPolygon`.

```
POST /api/analysis/trend
```
Same body as `/water-quality`. Returns only current status + trend direction (`improving` / `worsening` / `stable`).

```
POST /api/analysis/anomaly
```
```json
{
  "river": "Thames",
  "location": "London",
  "type": "Chemical",
  "severity": "High",
  "date": "2024-06-15",
  "description": "Industrial discharge detected upstream",
  "metrics": {
    "ph": 5.2,
    "dissolved_oxygen": 3.1,
    "turbidity": 40,
    "contaminant": "lead"
  }
}
```
Classifies the measurement as `normal` / `anomaly` / `critical` using EU Water Framework Directive thresholds. Returns `verdict`, `confidence`, `summary`, `risks`, and `recommendations`. OpenAI enriches the response if `OPENAI_API_KEY` is set, otherwise uses the deterministic heuristic.

```
POST /api/reports/generate
```
```json
{
  "event_id": "evt-001",
  "river": "Thames",
  "location": "London",
  "severity": "High",
  "type": "Chemical",
  "date": "2024-06-15",
  "description": "Industrial discharge upstream of intake.",
  "metrics": {
    "ph": 5.2,
    "dissolved_oxygen": 3.1,
    "turbidity": 40,
    "contaminant": "lead"
  }
}
```
Returns a downloadable PDF (`application/pdf`) styled as an EU Water Framework Directive compliance document. Includes a measurements table benchmarked against EU thresholds, AI executive summary (if OpenAI key is set), and recommended actions.

## WQI risk thresholds

The data-science pipeline uses a Waterly-calibrated WQI scale (range ~25вЂ“358):

| WQI | Risk level | Colour |
|-----|-----------|--------|
| в‰Ґ 200 | clean | `#10B981` |
| 150вЂ“199 | moderate | `#F59E0B` |
| 100вЂ“149 | high | `#EF4444` |
| < 100 | critical | `#7C2D12` |

## Frontend integration

CORS is open for `http://localhost:3000` (Next.js default).

For the map: fetch `GET /api/data/europe` and use it as a MapLibre GeoJSON source. Each feature's `risk_color` property maps directly to the `circle-color` paint property.
